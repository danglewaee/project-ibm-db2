Why this exists

The problem is not inventory mismatch. It is broken promises.

Who uses it

Sales ops, warehouse ops, and operations managers coordinating fulfillment across multiple warehouses.

Where it hurts

Reserved stock overlaps, partial shipments drift from reality, and cycle counts expose reconciliation gaps too late.

Why the model matters

Reservations, stock movements, shipment events, and audit trails have to remain internally consistent under transactional pressure.

Workflow surface

What the live service supports right now.

01

02

Technical truth

What this deployment proves, and what it does not claim.

Proved here

  • Spring Boot service can be packaged, built, and exposed as a public Cloud Run service.
  • Landing page reads live runtime metadata from the deployed backend.
  • Core order, reservation, shipment, cancellation, reconciliation, and audit slices exist in the service.

Proved earlier

  • Transactional workflow and schema were validated against live Db2 in Docker.
  • Flyway migrations and integration tests passed before this cloud demo was published.
  • The IBM Cloud Code Engine path is prepared, but not claimed as a completed paid-cloud deployment.
